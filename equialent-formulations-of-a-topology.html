1. Open Sets

Rather than look at a set with just elements, we instead look at a set with elements that approximate.

We now have two rules on what counts at approximating,

Approximations can get bigger and still be approximations forall lambda Ulambda, cuplambda Ulambda in A. If we have two approximations we can look at the intersection as another approximation.

Think about it like using a ruler with only cm on one side. Any measurement would look like Xcm +/- 1cm. Whenever we measure something there's always a bit of error. If we used infinately many approximations to hone in on a precise measurement, we wouldn't really get an approximation now would we? The infinite approximations would lead to a valid set but not a valid approximation.

Examples: (R, Teuclidean), (X, TCofinite), (X, TDiscrete), (X, TIndiscrete)

The sets are called open because given a set which approximates some element, x ∈ U, we can form a sequence an which approximates a \nin U. That is, the every element in an open set can be approximated from within the set but the sets are "leaky" in the sense that they can approximate other things.

2. Closed Sets

Now we have closed sets which have the opposite "leaky" property. Closed sets always have the property that if a point can be approximated from within the set, then the point was already within the set.

If you cannot escape any of the sets Fn then you cannot escape their intersection. Intersecting will not open a hole to escape frome.

If you cannot escape from two sets, then any sequence would be stuck infinately in at least one of the sets. The limit would then be from the set it has infinately many points in.

3. Closure Function

This next idea of being close does not use distinguished sets. Instead we have a function which takes any set from within X and ouputs all points which are touching the set.

The rules for what touches what are: S ⊂ cl(S) : S is close to itself cl[cl(S)] = cl(S) : The points close to the points close to S is close to S. cl(A cup B) = cl(A) cup cl(B) : If a point touches one of two sets, it must have been touching one of the sets. cl(∅) = ∅, the emptyset does not touch any points.

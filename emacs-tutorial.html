Emacs Tutorial

An opinionated collection of high level concepts for the Emacs text editor.

1. Emacs is an Emacs Lisp interpreter

This is the fundamental idea within the Zen of Emacs. Every key you press that gets sent to Emacs is a function which is called in Emacs. When one types, "a", this in fact calls upon the command self-insert-key with the parameter "a".

Workflows in Unix like systems involve using the shell to manipulate text and pipe said text into other programs. The method by which a user parses the text involves a sequence of regular expressions to get the right piece of data. In Emacs, data is not entirely text based, there is the string datatype, but most data are structured.

5. Flash Cards (Org-FC)

Spaced Repetition systems (SRS) are (often programatic) methods of recalling information. There is a nice SRS being actively developed called org-fc.

5.1. Math Definitions

Use a compact normal card. The Org heading is the definition name, then the body is the actual definition, "A X is Y if…".

5.2. Math Theorems

Double Sided Normal Card. Heading is name of theorem or some kind of summary of significance. One sub-heading is the full statement. Back sub-heading is the proof of the theorem. Put multiple cards for each of the steps of the proof. This makes the cards easier to compose and less likely to become a repeat card over time.

5.3. Program Syntax

Double sided card. Front states a question about what the syntax can accomplish. Back is the exact notation to use for the syntax.

5.4. Program Functions

Double sided card. Front is the function name and module path. Back is the arguments of the function and their purposes.

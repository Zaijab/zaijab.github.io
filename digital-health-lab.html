Machine learning models optimize for a certain loss function. While no model would outright have a preferential treatment for certain subpopulations before seeing the dataset, we note that some stages in the ML pipeline can introduce changes in the model which favor some subpopulations.

11.4. Information Geometry

In all of our ML projects, we are trying to fit a model to some dataset. In most circumstances, this entails solving an optimization problem (that is, doing calculus) over a set of parameters for a family of distributions. We may then apply principles of a more generalized calculus in higher dimensional spaces.

11.4.1. Gradient Methods

One very useful method to fit a distribution uses gradient descent methods. The rule we are all familiar with is:

\[ \theta_{t+1} \leftarrow \theta_{t} - \alpha \nabla_{\theta} L(\theta) \]

In words, the new parameters is an updated version of the old parameters plus some step in the direction of most descent.

The issue with this is that the distance between two distributions is not naturally expressed by Euclidean coordinates.

If we want to compare how different the blue vs orange distributions are, we see that the top plot (small variances) are somehow more different than the bottom plot (larger variances) because the bottom plot has more "overlap". However, if we just take the distance as the Euclidean distance between parameters, we get:

\begin{align} \sqrt{(\mu_{1} - \mu_{2})^{2} + (\sigma_{1} - \sigma_{2})^{2}} &= \sqrt{(1 - (-1))^{2} + (0.25 - 0.25)^{2}} = 2 \tag{Top Plot} \\ \sqrt{(\mu_{1} - \mu_{2})^{2} + (\sigma_{1} - \sigma_{2})^{2}} &= \sqrt{(1 - (-1))^{2} + (1 - 1)^{2}} = 2 \tag{Bottom Plot} \end{align}

Namely, the distance between the parameters are the same but we would like the distance to be different.

One way to quantify this difference is to look at how much information we lose by using one distribution as a model for the other. If there is less overlap, we lose more information by using one distribution over the other. Hence the information loss is higher. As a formula, we have,

\[ D_{\text{KL}}(P \ \| \ Q) = \int_{\Omega} \log \left( \frac{P(dx)}{ Q(dx)} \right) P(dx) \]

Using this as a "distance" (it is asymmetric an does not satisfy triangle inequality) we can define the "natural gradient" \( \tilde{\nabla} \) and use this natural gradient to do gradient descent.

\[ \theta_{t+1} \leftarrow \theta_{t} - \alpha \tilde{\nabla}_{\theta} L(\theta) \]

This natural gradient has a lot of nice properties (and some not so nice).

  • It requires the least steps to convergence
  • Invariant under sufficient statistics
  • Reparameterization invariant

Abstract: With recent advances in machine learning, many high stakes tasks are being done algorithmically. One wishes for the models deployed to not propagate discrimination and biases. Fair active learning has been proposed to reduce the cost and challenge of collecting an accurately labeled dataset and to build classifiers which behave fairly.

15.1.1. Fair Active Learning

There are 6 approaches to FAL in the paper by Anahideh: https://arxiv.org/pdf/2001.01796.pdf

The 6 approaches are given by a pair of choices

  1. Expected Unfairness Reduction
    1. Model agnostic expected value over all labels
    2. Using covariance as a proxy for fairness in General Linear Models (Efficient but only for simple models)
  2. Accuracy Fairness Optimization
    1. \( \alpha \)-Aggregate: Select a hyperparameter \( \alpha \in [0,1] \) which acts like a regularizing term
    2. FAL Nested: From a selection of high entropy points, we choose the one which maximizes unfairness reduction
    3. FAL Nested Append: If a point selected by FAL nested actually reduces the unfairness, then duplicate the point and add it into the labelled pool. Fit on the larger dataset (has more points than the budget actually allows).

Here we compare these 6 approaches to active learning and random labeling.

17. Differentiable Fairness

In Fairret: A Framework for Differentiable Fairness Regularization Terms, Buyl et al. introduces fairness evaluation within Pytorch, an autodiff environment. This allows users to incorporate fairness metrics within the loss function within a model during training and evaluating fairness metrics using probabilities.

Previously, we have been using the AIF360 which computes fairness metrics using predictions rather than prediction probabilities. We will use statistical_parity_difference as an example.

The idea behind statistical_parity_difference is to look at the positive rate of a classifier. This is important in credit loans and college entrances, where some believe that males and females should have an equal outcome of being selected. A system has some measured unfairness if

\[ \text{POSITIVE RATE}_{\text{FEMALES}} - \text{POSITIVE RATE}_{\text{MALES}} \]

AIF360 will perform the calculation by:

\[ \frac{\text{POSITIVE}_{\text{FEMALES}}}{\text{NUM\_FEMALES}} - \frac{\text{POSITIVE}_{\text{MALES}}}{\text{NUM\_MALES}} \]
